# Documentation Rules

이 문서는 Unhinged Students 프로젝트의 문서 관리 규칙을 정의합니다.

## CLAUDE.md 프로젝트 구조 갱신

프로젝트 구조가 변경될 때마다 CLAUDE.md를 최신 상태로 유지해야 합니다.

**갱신이 필요한 경우:**
- 새로운 파일이나 디렉토리를 추가했을 때
- 기존 파일의 역할이나 목적이 변경되었을 때
- 파일을 삭제하거나 이동했을 때

**작업 순서:**
1. 파일/디렉토리 변경 작업 수행
2. CLAUDE.md의 "프로젝트 구조" 섹션 찾기
3. 변경 사항 반영 (추가/수정/삭제)
4. 파일 설명이 정확한지 확인
5. 커밋 전 최종 검토

**예시:**
```markdown
## 📁 프로젝트 구조

```
Unhinged-Students/
├── src/
│   ├── game.js            # 메인 게임 루프
│   ├── character.js       # 캐릭터 시스템
│   └── newfeature.js      # [새로 추가된 파일 설명]
```

## 작업 현황은 TODO.md에만 기록

문서의 역할을 명확히 분리합니다.

**CLAUDE.md:**
- 프로젝트 개요 및 설명
- 프로젝트 구조
- 실행 방법
- 기술 스택
- 개발 가이드라인 및 규칙

**TODO.md:**
- 작업 진행 상태
- 완료된 작업 목록
- 대기 중인 작업
- 작업 브레이크다운

**금지 사항:**
- ❌ CLAUDE.md에 작업 현황 기록
- ❌ TODO.md에 프로젝트 구조나 실행 방법 기록
- ❌ 문서 간 내용 중복

## 기획서 참조

모든 개발 작업은 반드시 기획서를 참고해야 합니다.

**기획서 위치:** `docs/` 디렉토리

- **`docs/overview.md`** - 게임 전체 개요 및 스크린 플로우
- **`docs/characters.md`** - 캐릭터 상세 스펙
- **`docs/production.md`** - 린 개발 순서

**개발 시 주의사항:**
- 기능 구현 전 반드시 해당 기획서 섹션을 읽고 이해
- 기획서와 다른 방향으로 구현하지 않기
- 불명확한 부분은 사용자에게 질문하기
- `docs/production.md`의 개발 순서를 따르기
