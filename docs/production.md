# 🛠️ 미친 제자들 – 린 개발 순서 (Draft)

본 게임은 **린한 개발 방식**을 따른다.  
가장 재미의 핵심인 **게임 플레이 화면**부터 만들고,  
로컬 → 멀티 → 스크린 확장의 순서로 점진적으로 완성한다.

---

## 🎯 개발 원칙

- 화면은 **최소 기능(MVP)**부터 만든다
- 먼저 **재밌는지 확인**하고, 그 다음 확장한다
- 항상 **플레이 가능한 상태**를 유지한다
- 한 루프가 완성되기 전에는 다음 시스템을 만들지 않는다

---

## 0️⃣ 개발 준비 (세팅)

### 목표
- 게임이 실행되고, 화면이 뜬다

### 범위
- 웹 프로젝트 세팅
- 캔버스 / 렌더링 환경
- 키 입력 / 버튼 입력 테스트

---

## 1️⃣ 핵심 MVP – 게임 플레이 화면 (로컬)

### 🎮 대상 화면
- **6️⃣ 게임 플레이 화면**

### 목표
> “캐릭터를 띄워서 움직일 수 있다”

### 구현 내용
- 캐릭터 1개 표시
- 이동 가능
- 공격 버튼 작동
- 바닥에 샤드 생성
- 샤드 먹으면 레벨 숫자 증가
- 체력 감소 / 사망 처리
- 사망 후 리스폰

### 제외 (아직 안 함)
- 팀
- 멀티플레이
- 캐릭터 차이
- 보상
- UI 디자인

👉 **이 단계의 성공 기준**
- 혼자서도 계속 움직이고 때릴 수 있다
- “이거 게임 같다”라는 느낌이 든다

---

## 2️⃣ 로컬 팀전 확장 (5 vs 5, AI)

### 목표
> “팀으로 싸우는 느낌이 난다”

### 구현 내용
- 레드팀 / 블루팀 구분
- 팀별 캐릭터 5명 배치
- 플레이어 1명 + 나머지 AI
- AI 기본 이동 / 공격
- 킬 카운트

### 제외
- 멀티플레이
- 네트워크

👉 **이 단계의 성공 기준**
- 팀전 룰이 눈에 보인다
- 킬 수로 승패가 갈릴 것 같다는 감각이 있다

---

## 3️⃣ 로컬 게임 루프 완성

### 목표
> “한 판이 시작해서 끝난다”

### 구현 내용
- 제한 시간 타이머
- 게임 종료 조건
- 승패 판정
- 결과 데이터 계산 (킬 수)

👉 **이 단계의 성공 기준**
- 처음부터 끝까지 끊기지 않고 한 판 플레이 가능

---

## 4️⃣ 멀티플레이 전환 (최소)

### 목표
> “다른 사람이랑 같이 움직인다”

### 구현 내용
- 플레이어 위치 동기화
- 공격 / 사망 동기화
- 최소한의 네트워크 구조

### 주의
- 이 단계에서는 **UI / 보상 / 성장 없음**
- 오직 “같이 움직이고 싸우는지”만 확인

👉 **이 단계의 성공 기준**
- 두 명이 같은 맵에서 싸운다

---

## 5️⃣ 멀티 팀전 안정화

### 목표
> “멀티 5 vs 5가 돌아간다”

### 구현 내용
- 팀 자동 배정
- 리스폰 동기화
- 킬 카운트 동기화
- 시간 종료 동기화

👉 **이 단계의 성공 기준**
- 한 판 멀티 팀전이 끝까지 간다

---

## 6️⃣ 캐릭터 시스템 확장

### 목표
> “캐릭터마다 다르다”

### 구현 내용
- 외계인부터 구현
- 스킬 시스템
- 이후 캐릭터 순차 추가

### 순서
1. 외계인
2. 눈 돌아가는 사람
3. 카레 곰돌이
4. 헐크 언니
5. 찍찍찍찍찍
6. 선생님

👉 **항상**
- 새 캐릭터 추가 후
- 반드시 한 판 플레이 확인

---

## 7️⃣ 재화 & 보상 루프 추가

### 목표
> “플레이 이유가 생긴다”

### 구현 내용
- 루비 지급 (킬 수 기준)
- 다이아 드롭 / 손실
- 결과 화면

---

## 8️⃣ 스크린 확장

### 추가되는 화면
- 로비 화면
- 캐릭터 화면
- 접속 보상 화면
- 아이템 연출 화면

### 원칙
- **게임 플레이 루프가 깨지지 않게**
- 화면은 나중에 붙인다

---

## 9️⃣ 전체 루프 점검

### 최종 루프
접속
→ 보상
→ 로비
→ 캐릭터
→ 게임
→ 결과
→ 보상
→ 로비


### 체크 포인트
- 중간에 막히는 곳 없는가?
- 다시 한 판 하고 싶어지는가?

---

## 🔚 요약

> 먼저 **재밌는 핵심**을 만들고  
> 그 다음 **멀티를 붙이고**  
> 마지막에 **화면과 시스템을 채운다**
